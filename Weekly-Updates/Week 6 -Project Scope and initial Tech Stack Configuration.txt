Three-Tier Architecture

Presentation Layer (Frontend):
The client folder holds the React app, including all UI components, pages, and styles.

Displays forms, dashboards, and visual elements.

Sends user actions (login, signup, etc.) to the backend via API calls.

Application Layer (Backend):
The server folder manages business logic and routes.

Receives API requests from the frontend.

Communicates with MongoDB for data storage/retrieval.

Uses Express for routing and middleware.

Data Layer (Database):
MongoDB Atlas stores user records and app data securely.

Connection managed in server.js via Mongoose.

.env file securely holds connection credentials.

Key Files Explained

server.js: Starts the Express server and connects MongoDB.

authRoutes.js: Defines routes for user login/signup.

User.js: Mongoose schema for storing user information.

Login.jsx / Signup.jsx: Frontend components for user authentication.

auth.css: Handles styling for login/signup pages.

.env: Keeps sensitive data private (MongoDB URI, JWT secret).
